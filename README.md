# Placement Preparation Repository

A research-driven, company-wise placement preparation repository built to
maximise the probability of clearing the **Online Assessment (OA)** for the
exact roles applied to. Every document is generated only after researching the
company's actual assessment pattern, and **every question carries an
authenticity classification** so you always know how closely it maps to what
candidates have actually reported.

- **Version:** 1.1 (authenticity-classified, expanded, pattern-wise coding)
- **Generated:** July 2026
- **Format:** Professionally formatted, study-ready PDFs (each with a cover
  page, table of contents, an authenticity legend, page numbers and highlighted
  answers)

## Authenticity Classification

Every question and problem is labelled with one of four classifications. Labels
are assigned conservatively from research; authored practice questions are
**never** presented as real exam questions.

| Label | Meaning |
|---|---|
| **Verified Previous-Year** | The specific problem/pattern is documented from actual reported assessment experiences (e.g. BI3 "Happy Number" and "second highest salary"; Infosys subset-sum and split-array partitioning). |
| **Frequently Repeated** | The topic or pattern type is documented as recurring across multiple hiring cycles (e.g. Infosys number series, percentages, syllogisms). |
| **Community Reported** | A widely shared, role-relevant pattern reported across placement communities and forums. |
| **Research-Backed Company Style** | Authored to match the researched difficulty, style and pattern - practice, not a real past question. |

The label appears under each MCQ answer and beside each coding/SQL problem's
difficulty. A full legend page is included at the front of every PDF.

## Coding is Organised Pattern-wise

Coding PDFs group problems by algorithmic pattern so you drill the patterns that
recur in the assessment:

- **Infosys** (30 problems): Arrays, Strings, Hashing, Sliding Window, Two
  Pointers, Binary Search, Stack, Greedy, Dynamic Programming, Recursion &
  Backtracking, Math & Number Theory (DP and Greedy are emphasised for SP/DSE).

- **Tredence** (9 Python problems): counting, aggregation and data-manipulation
  patterns suited to an analyst role.

## Research & Verification Methodology

1. **Role-specific research first**, cross-verified across official pages,
   candidate interview experiences, and trusted placement platforms.
2. **Authenticity weighting** applied per question (see table above).
3. **Quality over quantity** - authentic, verified questions come first;
   company-style practice fills gaps without padding.
4. **Executed & verified code**: every Python solution is run and every SQL
   query is executed in SQLite; the printed output must match the stated sample
   output. (54 coding solutions + 15 SQL queries verified.)

## How to Use

1. Work through the **topic PDFs**, studying the pattern behind each answer.
2. For coding roles, drill the **pattern-wise Coding / Python / SQL** guides.
3. Finish with the **three Mock Tests** per company under timed conditions.
